# Restaurant Analytics & ML Dashboard

Проект выпускной квалификационной работы: **«Анализ эффективности ресторанного бизнеса в многомерных информационных системах с использованием метода машинного обучения»**.

Репозиторий содержит прототип аналитической системы для ресторанного бизнеса. Система генерирует локальное OLAP-хранилище данных по схеме «звезда», строит интерактивный веб-дашборд и обучает модели машинного обучения для прогнозирования ключевых бизнес-показателей.

## Возможности проекта

- создание локального аналитического хранилища в формате DuckDB;
- генерация демонстрационных данных по ресторанам, меню, клиентам, времени и продажам;
- модель данных по схеме «звезда»:
  - `Fact_Sales`;
  - `Dim_Date`;
  - `Dim_Time`;
  - `Dim_Restaurant`;
  - `Dim_Menu_Item`;
  - `Dim_Customer`;
- подготовленные SQL-представления для анализа:
  - `vw_daily_sales`;
  - `vw_menu_sales`;
  - `vw_restaurant_menu`;
  - `vw_customer_flow`;
- интерактивный Dash-дэшборд для многомерного анализа;
- фильтрация по периоду, ресторанам, сезонам, измерениям и метрикам;
- визуализации через Plotly: line, bar, area, pie, treemap;
- прогнозирование выручки, прибыли, количества проданных блюд и числа уникальных клиентов;
- сравнение моделей по метрикам MAE, RMSE, MAPE и R²;
- рекурсивный прогноз на будущий период.

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone <repository-url>
cd restaurant-analytics-ml
```

### 2. Создать виртуальное окружение

Для Windows:

```bash
python -m venv .venv
.venv\Scripts\activate
```

Для macOS/Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Создать базу данных

```bash
python src/create_restaurant_dw_duckdb.py --demo --db restaurant_dw.duckdb
```

По умолчанию база создаётся за период с `2024-01-01` по `2026-12-31`.

Можно указать собственный период:

```bash
python src/create_restaurant_dw_duckdb.py --demo --start-date 2024-01-01 --end-date 2026-12-31 --db restaurant_dw.duckdb
```

### 5. Запустить дэшборд

```bash
python src/restaurant_ml_dashboard.py --db restaurant_dw.duckdb
```

После запуска откройте в браузере:

```text
http://127.0.0.1:8050
```

## Использование скриптов

### Создание DuckDB-хранилища

```bash
python src/create_restaurant_dw_duckdb.py [--db PATH] [--start-date YYYY-MM-DD] [--end-date YYYY-MM-DD] [--demo]
```

Аргументы:

| Аргумент | Описание | Значение по умолчанию |
|---|---|---|
| `--db` | путь к файлу DuckDB | `restaurant_dw.duckdb` |
| `--start-date` | начальная дата календарного измерения | `2024-01-01` |
| `--end-date` | конечная дата календарного измерения | `2026-12-31` |
| `--demo` | заполнить хранилище демонстрационными данными | выключено |

> Для работы дэшборда нужна база с демонстрационными данными, поэтому при первом запуске используйте флаг `--demo`.

### Запуск Dash-дэшборда

```bash
python src/restaurant_ml_dashboard.py [--db PATH] [--host HOST] [--port PORT] [--debug]
```

Аргументы:

| Аргумент | Описание | Значение по умолчанию |
|---|---|---|
| `--db` | путь к файлу DuckDB | `restaurant_dw.duckdb` |
| `--host` | host Dash-сервера | `127.0.0.1` |
| `--port` | port Dash-сервера | `8050` |
| `--debug` | запуск в debug-режиме | выключено |

## Что показывает дэшборд

Дэшборд состоит из аналитического и прогнозного контура:

1. **Многомерный анализ**  
   Позволяет исследовать продажи, прибыль, скидки, средний чек, количество клиентов и другие показатели в разрезе ресторанов, дат, сезонов, блюд, категорий меню и клиентских сегментов.

2. **Прогнозирование**  
   Для выбранной целевой метрики строятся лаговые и календарные признаки, обучаются несколько моделей, сравнивается качество прогноза и формируется прогноз на будущие дни.

## Модели машинного обучения

В проекте используются следующие модели:

- `Baseline_Median`;
- `Ridge`;
- `RandomForest`;
- `ExtraTrees`;
- `HistGradientBoosting`.

Для оценки качества применяются:

- MAE;
- RMSE;
- MAPE;
- R².

Лучшей считается модель с минимальным значением RMSE.

## Примечания

- Файл базы данных `restaurant_dw.duckdb` не хранится в репозитории и создаётся локально.
- Папка виртуального окружения `.venv/` не должна попадать в Git.
- При первом запуске моделей дэшборд может загружаться дольше, так как выполняется обучение и расчёт метрик.
- Проект предназначен для демонстрации архитектуры аналитической системы и не использует реальные персональные данные клиентов.
